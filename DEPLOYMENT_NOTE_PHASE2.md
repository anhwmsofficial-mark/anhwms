# Phase 2 배포 메모 (Deployment Note)

> **배포 예정일**: 2026년 3월 7일  
> **대상 서비스**: ANH WMS (Admin/API)

---

## 1. 변경사항 요약
- **[운영 안정성]** 모든 API 요청에 추적 ID(`x-request-id`) 부여 및 감사 로그(Audit Log) 적재 시작.
- **[기능 정상화]** 작동하지 않던 입고 검수 기능 수정 (API 스키마 불일치 해결).
- **[보안 강화]** 관리자 검색 기능의 SQL Injection 취약점 패치 및 접근 제어 강화.
- **[신규 화면]** 관리자용 감사 로그 조회 메뉴(`/admin/audit-logs`) 추가.

## 2. 운영자 알림 (To Operations)
- **입고 검수**: 이제 '입고 검수' 버튼을 눌러 정상적으로 수량을 입력하고 저장할 수 있습니다.
- **로그 조회**: 시스템 변경 이력이 궁금하면 '설정 > 감사 로그' 메뉴를 확인하세요.
- **주의**: 기존에 잘못 생성된 테스트용 입고 데이터는 조회되지 않거나 에러가 날 수 있습니다. 신규 등록 건부터 정상 동작합니다.

## 3. 배포 후 필수 확인 (Smoke Test)
1. `/admin/audit-logs` 접속 시 로그 목록이 뜨는지 확인.
2. `/admin/inbound` 에서 입고 건 클릭 시 상세 정보가 뜨는지 확인.
3. 아무 입고 건이나 '검수' 시도 후 '저장'이 성공하는지 확인.

## 4. 롤백 포인트
만약 배포 후 입고 목록 접근이 안 되거나 500 에러가 쏟아지면 즉시 이전 버전으로 롤백합니다.
- **DB 롤백**: 코드 롤백만으로 해결되지 않는 경우, 추가된 `audit_logs` 컬럼은 무시해도 되므로 DB 롤백은 최후의 수단으로 보류.
