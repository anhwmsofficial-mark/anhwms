# 재고 신규제품 등록 운영 QA 체크리스트 (2026-02)

대상 화면: https://www.anhwms.com/inventory

## 1) 사전 준비

- [ ] 테스트 계정 권한: 재고관리/입고등록 모두 가능
- [ ] 테스트 고객사 2개 이상 준비 (예: A고객사, B고객사)
- [ ] 테스트 제품명 prefix 사용 (예: `TEST_20260210_...`)
- [ ] 시크릿 모드 1회 포함해 캐시 영향 점검
- [ ] 브라우저 기준: PC Chrome 우선 검증

## 2) 정상 케이스 (Happy Path)

### A. 신규 제품 등록 UI/필수값

- [ ] `+ 새 제품 추가` 클릭 시 입력 순서가 아래와 동일
  - 고객사 -> 제품명 -> 카테고리 -> 바코드 -> SKU(식별코드) -> 관리명 -> 사용자코드
- [ ] `SKU (식별코드)`가 필수(`*`)가 아님
- [ ] `제품DB번호 생성하기` 버튼 노출
- [ ] 초기 상태에서 `제품 추가하기` 비활성화(생성 전)

### B. 제품DB번호 생성 선행

- [ ] 고객사/카테고리 선택 후 생성 버튼 클릭 시 DB번호 생성
- [ ] 바코드 미입력 상태에서도 생성 시 자동 바코드 채움
- [ ] 생성 완료 후 `제품 추가하기` 버튼 활성화
- [ ] 생성된 DB번호는 읽기 전용으로 표시

### C. SKU 선택 입력

- [ ] SKU 미입력 상태로 등록 성공
- [ ] SKU 입력 상태로 등록 성공
- [ ] SKU 미입력 등록 시 목록에서 자동 SKU(`AUTO-...`) 확인

### D. 등록 후 재고 목록 반영

- [ ] 등록 성공 토스트/메시지 노출
- [ ] 모달 닫힘 후 재고 리스트 즉시 반영
- [ ] 검색창(제품명/SKU/바코드)으로 신규 제품 조회 가능

### E. 입고 검색 연계

- [ ] 입고 예정 등록 화면에서 동일 고객사 선택 후 신규 제품 검색 가능
- [ ] 동일 고객사에서 SKU/바코드 검색 가능
- [ ] 자동완성 선택 시 라인에 제품명/SKU/바코드 정상 반영

## 3) 예외 케이스 (Negative)

### A. 생성 전 검증

- [ ] 고객사 미선택 + 생성 클릭 시 안내 메시지 노출
- [ ] 카테고리 미선택 + 생성 클릭 시 안내 메시지 노출

### B. 생성 후 값 변경

- [ ] DB번호 생성 후 고객사 변경 시 등록 버튼 재비활성화
- [ ] DB번호 생성 후 카테고리/바코드 변경 시 재생성 필요 상태 전환
- [ ] 재생성 후에만 등록 가능

### C. 중복/무결성

- [ ] 생성 버튼 연속 클릭 시 충돌 없이 처리
- [ ] DB번호 중복이 발생하지 않음(재시도 로직 확인)

### D. 오류/복구

- [ ] 생성 API 실패 시 오류 메시지 노출
- [ ] 실패 시 생성 버튼 로딩 상태 정상 해제
- [ ] 실패 상태에서 등록 버튼이 잘못 활성화되지 않음

## 4) 회귀(Regression)

- [ ] 기존 제품 수정 모달(저장/취소) 정상 동작
- [ ] 재고 목록 페이징/검색/카테고리 필터 정상
- [ ] 입고 신규/수정 화면 상품 자동완성 성능 저하 없음
- [ ] 바코드 스캔 후 품목 자동추가 정상
- [ ] 엑셀 업로드 SKU 매핑/품목 생성 흐름 정상

## 5) 운영 승인 기준 (Go / No-Go)

### Go

- [ ] 정상 케이스 100% 통과
- [ ] 예외 케이스에서 안내/복구 흐름 정상
- [ ] 입고 검색 누락 재현 불가

### No-Go

- [ ] DB번호 생성 없이 등록 가능한 우회 케이스 존재
- [ ] 특정 고객사에서 입고 검색 누락 재현
- [ ] 등록 성공 후 목록 미노출/지연 지속

## 6) 운영 반영 후 당일 모니터링

- [ ] 신규 제품 생성 대비 입고 검색 실패 문의 건수 확인
- [ ] `generate-db-no` API 에러율/응답시간 확인
- [ ] 제약조건 에러(`product_db_no`, `sku`) 발생 여부 확인
- [ ] 사용자 피드백: 생성 버튼 선행 절차 이해도 확인
