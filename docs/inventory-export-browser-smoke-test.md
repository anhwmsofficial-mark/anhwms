# 재고 엑셀 시스템 브라우저 스모크 테스트

## 목적
- 운영 적용 직후 브라우저에서 5~10분 안에 핵심 기능만 빠르게 확인합니다.

## 사전 조건
- 마이그레이션 적용 완료
- 관리자 계정 로그인 가능
- 최소 1개 이상 품목, 창고, 템플릿 존재

## 테스트 경로
- 재고 메인: `/inventory`
- 템플릿 설정: `/admin/inventory/export-templates`

## 시나리오 1. 템플릿 설정 페이지 열림
1. `/admin/inventory/export-templates` 접속
2. 화면이 에러 없이 로드되는지 확인
3. 좌측 목록에 `inventory-default` 또는 기존 템플릿이 보이는지 확인

기대 결과:
- 목록/편집 화면 정상 표시

## 시나리오 2. 템플릿 수정 저장
1. 기존 템플릿 선택
2. 트랜잭션 컬럼 하나 체크 해제
3. 헤더명 하나를 눈에 띄는 값으로 수정
4. `템플릿 저장` 클릭
5. 새로고침 후 저장값 유지 확인

기대 결과:
- 성공 토스트 표시
- 체크 상태와 헤더명이 유지됨

## 시나리오 3. 재고 메인 화면 조회
1. `/inventory` 접속
2. 기준일 선택
3. 업체 선택 또는 전체 조회
4. 그리드가 정상 렌더링되는지 확인

기대 결과:
- `관리명`, `전일재고`, 트랜잭션 컬럼들, `현재고` 노출

## 시나리오 4. 열 설정 반영
1. `/inventory`에서 `열 설정` 클릭
2. `파손` 또는 `샘플(-)` 컬럼 체크 해제
3. 적용 후 그리드에서 해당 열이 사라졌는지 확인

기대 결과:
- 화면에서 해당 컬럼 미노출

## 시나리오 5. 재고 변동 입력
1. 임의 품목의 `파손` 셀 클릭
2. 창고 선택
3. 수량 `1`
4. 사유 입력 후 저장
5. 현재고가 즉시 반영되는지 확인

기대 결과:
- 성공 토스트 표시
- 현재고가 1 감소

## 시나리오 6. 음수 재고 방지
1. 현재고보다 큰 수량으로 차감성 이벤트 입력 시도
2. 저장 버튼 클릭

기대 결과:
- 오류 메시지 표시
- 저장되지 않음

## 시나리오 7. 업체용 엑셀 다운로드
1. `/inventory`에서 템플릿 선택
2. `업체용 엑셀 다운로드` 클릭
3. 파일 저장
4. 엑셀 열기

기대 결과:
- 파일 다운로드 성공
- 시트명 정상
- 숨긴 컬럼이 실제 파일에서도 제외
- 수정한 헤더명이 파일에 반영

## 시나리오 8. 최종 회귀 체크
1. 다시 `/inventory` 새로고침
2. 방금 입력한 트랜잭션이 그리드에 반영되어 있는지 확인
3. 다시 다운로드해서 같은 내용이 반영되는지 확인

기대 결과:
- 화면과 엑셀 값 일치

## 실패 시 우선 확인
- 템플릿 저장 안 됨:
  `export_templates`, `export_template_columns` 테이블 존재 여부 확인
- 재고 입력 안 됨:
  창고/상품/권한/음수재고 검증 메시지 확인
- 다운로드 안 됨:
  템플릿 선택 여부, `inventory-default` 존재 여부 확인
- 화면이 비어 있음:
  `products`, `inventory_snapshot`, `inventory_ledger` 데이터 존재 여부 확인

## 권장 최소 통과 기준
- 템플릿 저장 성공
- 재고 변동 1건 저장 성공
- 음수 재고 차단 동작 확인
- 엑셀 다운로드 성공
- 화면과 다운로드 파일 값 일치
